# AGENTS.md

本文件约束 LLM 在本仓库中的编码、文档和配置维护行为。冲突时按优先级执行：系统/开发者指令 > 本文件 > `CLAUDE.md` > 其它项目文档 > 局部习惯。

## 1. 硬性门禁

违反以下任意规则时，必须停止并修正：

1. **NEVER 幻觉引用**：函数、类型、字段、文件、IPC channel、工具名、配置项和文档路径必须先通过搜索或阅读确认存在。找不到依据就继续搜索；搜索无果则向用户确认。
2. **MUST 先思考再修改**：任何代码、配置、契约性文档或项目规则修改前，必须先输出第 2 节的 `[Pre-Flight Manifest]`。Manifest 前只允许只读搜索和文件查看。
3. **NEVER 超范围修改**：只处理用户要求和直接依赖的文件。不顺手修无关问题，不做未请求的重构、格式化或目录整理。
4. **MUST 不确定时暂停**：遇到会影响核心逻辑、数据格式、安全边界或用户工作流的不确定点，必须先确认。
5. **NEVER 吞掉错误**：不能为了继续流程而忽略真实失败。IPC、工具、持久化、构建和文档验证失败都必须可追踪、可定位。
6. **NEVER 引入真实密钥**：代码、测试、文档、示例和提交说明中不得写入真实 API key、证书、token 或本机敏感路径。
