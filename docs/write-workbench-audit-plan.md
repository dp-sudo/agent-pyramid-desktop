# 写作工作台现状审查与修复规划

本文档沉淀本次对写作工作台前端模块的只读审查结论，并作为后续逐项修复的执行输入。本文档记录的是待处理问题和修复规划，不代表当前 UI 权威状态；当前状态仍以项目架构、UI 设计、UI 布局、数据模型和 IPC 契约文档为准。

## 1. 审查范围与事实依据

本次审查覆盖 8 个区域和 1 个联动项：

| 区域 | 覆盖内容 |
| --- | --- |
| 界面布局 | 写作工作台整体结构、栏位划分、留白、响应式 |
| UI 细节 | 间距、圆角、阴影、图标、按钮状态 |
| 界面色调 | 主题切换、跟随系统、明暗双套、对比度 |

| 文本信息的排序与显示色调 | 时间线顺序、流式输出、视觉权重 |

## 2. 模块概览

写作工作台当前由以下模块共同构成：

| 模块 | 当前职责 |
| --- | --- |
| 写作路由外壳 | 挂载写作工作区，并显示浮动错误提示 |
| 写作工作区 | 管理工作区选择、Markdown 文件列表、搜索、打开文件、编辑内容、自动保存、内联补全 |
| 编辑面板 | 承载 Markdown 文本编辑区、补全幽灵文本、保存状态栏 |
| 写作助手面板 | 展示写作线程近期对话项，嵌入写作版输入器 |
| 共享输入器 | 写作模式隐藏附件、模型选择、计划和目标入口，仅保留文本发送与中断 |
| 消息渲染组件 | 渲染助手 Markdown、代码块、复制、表格、图片、安全链接、审批和工具详情 |
| 设置页相关控制 | 管理写作默认模型、工具可见性、审批展示、代码块折叠、主题与布局偏好 |
| 主进程写作文件服务 | 负责 Markdown 文件列表、读取、保存和本地补全，执行工作区路径边界校验 |

与主进程通信点：

| 通信点 | 用途 | 当前结论 |
| --- | --- | --- |
| 工作区目录选择 | 选择本地工作区目录 | 已通过预加载 API 进入主进程 |
| 写作文件列表 | 列出工作区内 Markdown 文件 | 已限制 Markdown 扩展名和跳过目录 |
| 写作文件读取 | 读取当前 Markdown 文件 | 已使用严格 UTF-8 读取，失败可见 |
| 写作文件保存 | 保存当前 Markdown 文件 | 已经走主进程路径边界校验 |
| 写作内联补全 | 提供本地 Markdown 模式续写 | 当前是本地规则补全，不是 LLM 补全 |
| 线程选择与创建 | 为工作区绑定写作线程 | 已按工作区选择或创建写作线程 |
| 回合开始与中断 | 发送写作助手请求并中断运行中回合 | 已接入统一运行时 |
| 运行时事件订阅 | 接收助手回复、系统消息、审批、工具和错误事件 | 订阅存在，但写作助手展示过滤过多 |
| 运行时偏好 | 支撑写作默认模型、工具可见性、审批展示偏好 | 设置与持久化存在，部分写作路线消费不完整 |

当前已实现的核心闭环：

| 闭环 | 当前行为 |
| --- | --- |
| 打开工作区 | 选择目录，选择或创建写作线程，设置工作区，加载 Markdown 文件列表 |
| 打开文件 | 保存当前脏文档，读取目标文件，防止旧响应覆盖新选择 |
| 编辑保存 | 文档变更后自动保存或手动保存，保存成功显示已保存，失败显示错误 |
| 切换路线 | 离开写作路线前保存脏文档，失败则留在写作工作台 |
| 写作助手 | 显式输入写作请求，只携带当前文件路径和保存状态，不镜像全文 |
| 错误反馈 | 写作局部错误进入状态栏，全局运行时错误进入浮动提示 |

## 3. 缺陷清单

| 序号 | 区域 | 问题描述 | 风险等级 | 触发场景 | 期望行为 | 当前行为 | 涉及模块 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| D-01 | 界面布局 | 写作侧栏复用全局左栏宽度，但写作路线没有直接拖拽或键盘调宽入口 | 中 | 用户在写作路线管理大量文件 | 能在写作路线直接调整文件列表宽度 | 只能受代码路线或设置影响 | 写作工作区、全局布局状态 |
| D-02 | UI 细节 | 保存按钮有禁用逻辑，但缺少清晰禁用视觉 | 中 | 文件无变化、正在保存、未打开文件 | 禁用状态应明显不可操作 | 视觉仍接近可点击强调按钮 | 编辑面板、按钮样式 |
| D-03 | UI 细节 | 状态栏错误与保存按钮同处横向行，长错误缺少清晰换行或截断策略 | 中 | 文件读写失败且错误信息较长 | 错误可读且按钮不被挤压 | 长错误可能压缩状态栏布局 | 编辑面板、状态栏 |
| D-04 | 界面色调 | 文件元信息和差异摘要中出现异常分隔字符，并被测试固化 | 低 | 查看文件列表元信息或审批差异摘要 | 使用本地化、语义明确的分隔符 | 显示异常字符 | 文件列表元信息、差异摘要 |
| D-05 | 交互逻辑 | 写作助手消息区每次消息变化都会滚到底部 | 中 | 用户阅读较早回复时后台继续流式输出 | 用户离底部较远时不抢滚动 | 更新会拉回底部 | 写作助手面板 |
| D-06 | 交互逻辑 | 写作内联补全固定显示在编辑区底部，不贴近光标 | 中 | 长文档中触发列表或引用续写 | 补全应贴近当前输入位置 | 补全固定覆盖底部区域 | 编辑面板、补全展示 |
| D-07 | 文档渲染 | 当前 Markdown 文档只有源码编辑，没有预览或分屏预览 | 中 | 撰写标题、表格、代码块、任务列表 | 能快速检查最终排版 | 只能查看源码文本 | 编辑面板 |
| D-08 | 文本排序 | 写作助手只显示用户、助手和系统消息，过滤推理、工具、审批和计划过程项 | 高 | 写作线程触发工具、审批、推理或计划项 | 用户能看到完整运行过程和状态 | 关键过程项不显示 | 写作助手消息投影 |
| D-09 | 文本排序 | 写作助手没有按回合分组，也没有工作过程折叠区 | 中 | 连续多轮写作请求 | 每轮输入、过程、最终回答层级清楚 | 近期消息平铺展示 | 写作助手面板 |
| D-10 | 设置完整性 | 审批展示偏好在代码路线有消费点，但写作路线缺少审批面板 | 高 | 写作路线开启需审批工具 | 可显示审批、差异、提交中状态 | 写作路线无法完成审批操作 | 设置页、写作助手、审批链路 |
| D-11 | 工具隔离 | 设置页允许显式开启写作路线的危险工具，但写作工作台没有二次确认入口 | 高 | 用户开启写作路线写文件或命令工具后模型请求执行 | 写作路线展示危险操作二次确认 | 运行时可请求审批，但写作 UI 不呈现操作面 | 工具权限、写作助手 |
| D-12 | 体验联动 | 写作助手上下文只包含当前文件路径和保存状态，不包含选中文本或光标附近片段 | 中 | 用户要求改写当前段落或润色选中内容 | 可携带精确局部上下文且不泄露全文 | 需要用户手动描述目标片段 | 写作工作区、助手请求构造 |

## 4. 已确认无缺陷或暂不作为缺陷处理的能力

| 能力 | 当前结论 |
| --- | --- |
| 写作默认模型 | 设置可读、可写、持久化，并被运行时消费 |
| 默认工具隔离 | 写作线程默认隐藏并拒绝代码写入和命令类工具 |
| 写作文件服务安全边界 | 文件列表、读取和保存由主进程执行，限制 Markdown 文件并做工作区路径边界校验 |
| 写作助手不镜像全文 | 助手请求只携带显式输入、当前文件路径和保存状态，避免默认泄露整篇文档 |
| 助手 Markdown 回复渲染 | 助手回复复用共享 Markdown 渲染，具备代码块、复制、表格、图片和安全链接处理 |
| 离开写作前保存 | 切换文件、刷新、切换工作区、离开写作路线前会尝试保存脏文档 |
| 打开文件乱序保护 | 慢响应不会覆盖用户后续选择的文件 |

## 8. 修复规划

### 第 1 阶段：工具与审批安全闭环

目标：修复写作路线开启危险工具后无审批操作面的 P0 风险。

| 任务 | 覆盖问题 | 验收标准 |
| --- | --- | --- |
| 在写作路线显示未决审批 | D-10、D-11、H-01、H-05 | 写作线程触发审批时，用户能在写作路线看到审批卡片 |
| 接入允许和拒绝操作 | D-10、D-11 | 点击允许或拒绝后进入提交中状态，并禁用双按钮 |
| 显示审批差异预览 | D-10、H-02 | 写作路线遵守审批差异默认展开设置 |
| 保留审批审计记录 | D-08、D-09 | 审批项在写作助手历史中可追踪 |

建议验证：

| 验证项 | 方式 |
| --- | --- |
| 写作路线未决审批可见 | 构造写作线程审批项并渲染写作助手 |
| 允许和拒绝只提交一次 | 验证提交中状态和按钮禁用 |
| 偏好项生效 | 切换审批差异默认展开和自动滚动设置后观察写作路线 |

### 第 2 阶段：写作助手时间线与过程展示

目标：让写作助手从简单近期消息列表升级为可追踪的写作线程时间线。

| 任务 | 覆盖问题 | 验收标准 |
| --- | --- | --- |
| 显示推理、工具、审批、计划等过程项 | D-08、H-02 | 写作助手不再过滤关键运行时项 |
| 按回合分组展示 | D-09 | 每轮用户输入、工作过程、最终回复层级清晰 |
| 失败工具保持可见 | D-08 | 即使隐藏成功只读工具，失败工具仍可见 |
| 保持助手回复 Markdown 渲染质量 | D-07、已确认能力 | 代码块、复制、表格、图片和安全链接不退化 |

建议验证：

| 验证项 | 方式 |
| --- | --- |
| 多轮顺序稳定 | 构造多轮消息项并验证显示顺序 |
| 过程区折叠状态稳定 | 流式更新不重置用户手动折叠选择 |
| 工具失败可见 | 设置隐藏只读工具后，失败工具仍显示 |

### 第 3 阶段：写作助手滚动与状态反馈

目标：修复高频体验问题，避免抢滚动和状态混乱。

| 任务 | 覆盖问题 | 验收标准 |
| --- | --- | --- |
| 写作助手采用接近底部才自动跟随的滚动策略 | D-05 | 用户阅读历史内容时，流式输出不抢滚动 |
| 增加跳转到最新回复入口 | D-05 | 用户离底部时可一键回到最新内容 |
| 保存按钮增加明确禁用态 | D-02、H-03 | 不可保存时视觉和指针状态明确 |
| 状态栏错误支持长文本展示 | D-03 | 长错误不挤压保存按钮，完整错误可读 |

建议验证：

| 验证项 | 方式 |
| --- | --- |
| 滚动不抢焦 | 模拟消息追加时用户不在底部 |
| 禁用态明确 | 渲染保存按钮各状态 |
| 长错误可读 | 注入长错误文本检查布局 |

### 第 4 阶段：Markdown 文档显示质量

目标：提升写作核心编辑体验。

| 任务 | 覆盖问题 | 验收标准 |
| --- | --- | --- |

| 预览复用安全 Markdown 渲染规则 | D-07 | 链接和图片同样遵守安全白名单 |
| 补全显示贴近输入上下文 | D-06、H-04 | 长文档中补全不固定在底部遮挡内容 |
| 提供接受和取消补全的可见反馈 | D-06 | Tab 接受和 Escape 取消行为清晰 |

建议验证：

| 验证项 | 方式 |
| --- | --- |
| Markdown 元素渲染 | 使用标题、表格、代码块、任务列表样例 |
| 不安全链接不渲染为可点击 | 构造不安全协议样例 |
| 补全不遮挡正文 | 长文档底部和中部输入场景 |

### 第 5 阶段：写作上下文联动

目标：增强助手对当前写作内容的理解，同时保持不默认泄露全文的边界。

| 任务 | 覆盖问题 | 验收标准 |
| --- | --- | --- |
| 支持携带选中文本作为显式上下文 | D-12 | 用户选中文本后发送请求，助手上下文包含选区 |
| 支持携带光标附近有限片段 | D-12 | 没有选区时可携带受限长度的邻近片段 |
| 在发送内容中标明上下文来源 | D-12 | 助手能区分用户请求、选区和邻近片段 |
| 保留不镜像全文原则 | 已确认能力 | 默认不把完整文档放入助手请求 |

建议验证：

| 验证项 | 方式 |
| --- | --- |
| 选区上下文 | 选中段落后发送请求，载荷包含选区 |
| 邻近片段边界 | 超长文档只携带有限片段 |
| 空选区行为 | 无选区时仍可发送普通写作请求 |

## 10. 每轮修复的通用验收要求

每轮实现前应重新执行本仓库要求的修改前判断，并只修改当轮直接相关文件。每轮完成后至少满足：

| 验收项 | 要求 |
| --- | --- |
| 变更范围 | 只覆盖当轮目标问题 |
| 事实依据 | 新引用的字段、类型、函数、文件和接口均先搜索确认存在 |
| 错误处理 | 不吞掉真实错误，用户可见失败必须保持可追踪 |
| 设置闭环 | 涉及设置项时确认可读、可写、生效、反馈和持久化 |
| 工具安全 | 涉及工具和审批时确认危险操作有二次确认 |
| 文档同步 | 修改 UI、IPC、运行时偏好或数据契约时同步对应项目文档 |
| 自动验证 | 代码改动后运行类型检查、测试和构建 |
